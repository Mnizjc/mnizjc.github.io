hexo博客搭建踩坑实录

发表于2022-10-12|更新于2025-08-10|Hexo
|阅读量:

说实话不踩坑是真学习不到东西的,教训总比经验让人记忆深刻

本来就想着如果自己注册个域名,搭个服务器跑这跑那的挺麻烦,所以我就选择了把这玩意部署到github上(纯懒,纯想白嫖,想寄生),但是像github pages,git这些东西我还真是第一次用,纯小白一个,才踩了这些个坑

原本就不需要花多少功夫,但是我还是因为一些破事忙了一天

实际上这些错误都是我犯糊涂所导致的

先是一个小问题:hexo上的markdown语法规则/hexo标题

这一步我还没部署到github上,还是在本地

我直接我直接

我直接没有直接嘛

不去打开那个hello-world看看还真不知道

hexo每次新建一个post都会在开头加这么个东西:

如果我再多加一个一级标题并且和这里的title内容不一样,那结果就是

你可能花了那么久写的一篇文章,到时候后台简单报个错,前端直接当你啥都没干过一样

说实话这东西挺让人无语的

不过既然一级标题title已经解决了,那二级标题开头就没事了

执行hexo d时出现的问题

为了不让自己把这件事忘了,我在push完所有东西之后,没有直接把终端关掉,而是赶快把上面的东西都记下来

实际上部署需要的工作很简单:执行hexo clear && hexo d就完了

不过我这里用的是powershell所以命令应该是hexo clear ; hexo d,中间是一个分号

但是我还是遇到了这些问题

ssh连接github不成功

本来大多数教程都是用的ssh连接github,但是我不管怎么ssh,总是

我就直接放弃ssh了

代理不成功

地域问题,我大西域不好访问github,但是我发现了一个工具:dev-sidecar,它连着访问github,git,npm,pip的代理都包了,这点确实舒服,就在我开着边车快乐运行git的时候

代理还出错了?

之前遇到过这个情况,把ssl验证去了,sidecar的ssl验证关了就行了

最让我难受的一个

有几句话说的好

「磨刀不误砍柴工」

「你和人起争执,你们还可以争论到底是谁的错;但是你和电脑起争执,不用怀疑了,就是你的错」

实际上这个错误已经出现过很多次了,但是我当时就是不知道我到底错那了

之前说我已经放弃ssh的方法了,直接用https,也就是说我在push之前要进行一次验证,输入username和password,但是我每次都是直接输入的我自己登录github的账号和密码,所以每次验证都会报错一次

这个错误从早上开始一直伴随我到下午,我都快「爱上」它了

hexo文档也没说到这一点,以至于我一直蒙在鼓里,没办法,这真是我第一次用git

并且我是真的瞎到光去找错在网上查,从来不多看一眼错误提示,实际上人家已经给你在上面写的清清楚楚了

真的就和没用过git和github一样…

打开之后我才发现应该这么做

When you , , , or to a remote repository using HTTPS URLs on the command line, Git will ask for your GitHub username and password. When Git prompts you for your password, enter your personal access token (PAT)

我才注意到有个东西叫pat,也就是说,账户名还是账户名,但是密码应该输入的是pat.

这是我最后一个坑,当我看到这条消息

并再次打开之后发现已经被替换成hexo的页面时,我终于是长舒了一口气,自己有问题还得自己白花了这么长时间.还是得学习

安装主题出问题

我想着去安装一个主题再把公式渲染器也装上,但是装上主题再配置好后发现主页直接没了,只显示这些东西

查了资料后发现,如果现实这个玩意,那就是说没装hexo-renderer-swig,那就去装一个呗

原来npm的install命令可以直接简化成i,uninstall命令简化成un

安装后问题消失

tex公式渲染问题

我不把那三篇搬过来我还真不知道,hexo原生不支持tex公式渲染.那就去装一个
