CloudFlare+Github Pages实现博客完全体

发表于2025-07-22|更新于2026-02-23|Hexo
|浏览量:

一直觉得这东西正文访问速度还算不错，就是图片加载速度台恶心了，所以想试试CloudFlare，顺便把博客的域名也换一下，顺便把博客的图片换个地方托管，这样访问速度应该会快很多。

之前没有找到方便好用的图床，所以一直用的jsDelivr，但众所周知，jsDelivr被吊销许可已经有一段时间了，虽然搭建这段时间好像挺好用的，但终究不是长久之计，所以还是得找个新的图床。

实际上现在想想导致我这东西看上去加载速度慢的本质原因就是图片始终加载特别慢，通常文章正文早就加载完了，而图片还在一点点往下走，直观感觉就是这个网站有点摇摇欲坠，感觉马上就要被墙了。

早就有一段时间像搞一下加速的事了。之前看有说能用CloudFlare CDN加速，但是要托管需要域名，孩子穷，不是说.xyz之类的域名用不起，我承认这东西现在价格确实挺便宜的，而是我不知道我什么时候能把这东西搞定，如果没搞定，那这域名就白买了，所以一直想整个免费的。但是众所不周知（因为我还不太清楚），.tk等域名在freenom上已经名存实亡了，直接不允许用户注册（虽然我以前想注册的时候也没有注册成功过，但是一直以为只是因为被墙了，没想到是彻底没了）。

不过好在我这两天刷到了一个免费域名的好东西，去试了一下，还真注册成功了。挺爽的，等会正文中讨论。

我的域名注册流程和CloudFlare配置流程参考了BV1uAVLzCEmH，当然我也是从这里知道了这个域名能在这里注册。

域名

这两天刷到一个二级域名能拿来当顶级域名用的好东西：https://nic.us.kg/。金坷垃好处都有啥，谁说对了就给他！自从免费顶级域基本灭绝之后，我就准备去找好看点的二级域。也不是说我之前找到的一点点结果都没有了，但是基本都是如.eu.org这样的自从我申请之后等了大概快两个星期了还没有通过（那种上古时代的页面真让人印象深刻，并且它是真的「上古」，since 1996的分量你懂吗），或者就是要这要那的信息有点多了，有点麻烦，遂放弃。直到刷到这东西，那可真是太好了。

写的时候一直开着codegeex，经常会给点什么小「建议」,这种特性虽然好，但是架不住它经常给点要么戾气特别大要么特别抽象的建议（此时它的建议是「还是关了」，这下更抽象了）。智谱！你给你们家的模型都喂了什么东西啊！

还有一点我差点忘了

因为域名是用CNAME绑定到Github Pages的，而hexo g会直接覆盖仓库根目录，所以每次hexo d会直接把CNAME文件删掉，导致每次更新完域名都会丢掉。

解决方法是在源码仓库的source文件夹下直接把CNAME文件放进去，这样每次hexo g会直接把source中的内容复制到pages仓库根目录，这样就不会丢掉了。
